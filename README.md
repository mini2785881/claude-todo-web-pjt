# 오늘의 할 일 (Todo List Web)

프레임워크나 빌드 도구 없이 순수 HTML/CSS/JavaScript로 만든 개인용 할 일 관리 웹 앱입니다.

## 실행 방법

`index.html` 파일을 더블클릭해서 브라우저로 열면 바로 실행됩니다. 별도 서버나 설치 과정이 필요 없습니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 화면 구조 (헤더/입력폼/필터바/할 일 목록) |
| `style.css` | 카드형 레이아웃, 카테고리 색상, hover/transition, 반응형 스타일 |
| `app.js` | 할 일 데이터 관리, 렌더링, 이벤트 처리, localStorage 저장/로드 |

## 사용법

- **추가**: 입력창에 할 일을 입력하고 카테고리를 선택한 뒤 `추가` 버튼 또는 Enter
- **완료 체크**: 항목 앞 체크박스 클릭 → 취소선 표시, 목록 하단으로 이동
- **수정**: ✎ 버튼 클릭 → 텍스트를 인라인으로 수정 → Enter 또는 `확인` 버튼으로 저장, Esc로 취소
- **삭제**: 🗑 버튼 클릭 → 확인창(confirm)에서 승인 시 삭제
- **필터**: 상단 필터 버튼(전체/개인/공부/업무/취미)으로 카테고리별 보기 전환
- **진행률**: 전체 할 일 대비 완료 비율이 상단 진행률 바 + `n/m 완료 (%)` 텍스트로 표시 (필터와 무관하게 전체 기준)

## 데이터 저장

- 브라우저 `localStorage`의 `todos` 키에 JSON 배열로 저장됩니다.
- 할 일이 추가/수정/삭제/완료 토글될 때마다 자동 저장되며, 새로고침해도 데이터가 유지됩니다.
- 데이터 모델: `{ id, text, category, completed, createdAt }`

## 최종 점검 결과

아래 항목을 헤드리스 브라우저(jsdom)로 자동 검증했습니다 (18개 시나리오 전체 통과).

- [x] 할 일 추가/수정/삭제 정상 동작 (빈 텍스트 추가 방지, 삭제 confirm 취소 시 유지 포함)
- [x] 완료 체크 토글 및 취소선/흐림 시각적 구분, 완료 항목 목록 하단 정렬
- [x] 카테고리 분류 및 필터 버튼 클릭 시 정확한 필터링 + active 스타일
- [x] 진행률(완료/전체, %)이 정확히 계산되어 텍스트와 프로그레스 바에 반영
- [x] 새로고침(재로드) 후에도 목록/완료 상태가 그대로 유지
- [x] 손상된 localStorage 값에도 예외 없이 빈 목록으로 안전하게 처리
- [x] 외부 프레임워크/빌드 도구 없이 `index.html` 실행만으로 동작

## 참고

- 최신 Chrome/Edge 기준으로 테스트되었습니다.
- 브라우저 로컬 저장소를 사용하므로, 다른 브라우저나 시크릿 모드로 열면 데이터가 공유되지 않습니다.
